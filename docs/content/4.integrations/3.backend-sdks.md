## Official Sockudo Server SDKs

Sockudo provides official server SDKs for 8 languages. All support `idempotency_key` for safe publish retries.

## Compatibility Contract

Sockudo server SDKs follow the same compatibility split as the realtime protocols:

- **V1** is the Pusher-compatible contract.
- **V2** is the Sockudo-native contract.

In practice, this means:

- Existing Pusher server SDKs can continue to target Sockudo for basic event publishing, channel auth, and webhook verification.
- Sockudo server SDKs are the recommended path when you want Sockudo-specific extensions such as `idempotency_key` and future V2-native features.
- Sockudo server SDKs should stay behaviorally close to Pusher-compatible semantics for V1 workflows, but Sockudo-only features are additive and should not be described as part of the Pusher contract.

## Usage Examples

::code-group

```js [Node.js]
const { Sockudo } = require("sockudo");

const sockudo = new Sockudo({
  appId: "app-id",
  key: "app-key",
  secret: "app-secret",
  host: "127.0.0.1",
  port: 6001,
  useTLS: false,
});

// Publish an event
await sockudo.trigger("orders", "order.updated", { id: 42 });

// Publish with idempotency key (safe retries)
await sockudo.trigger("orders", "order.updated", { id: 42 }, {
  idempotency_key: "unique-request-id-123"
});

// Batch publish
await sockudo.triggerBatch([
  { channel: "orders", name: "order.created", data: { id: 1 } },
  { channel: "orders", name: "order.created", data: { id: 2 } },
]);
```

```php [PHP]
use Sockudo\Sockudo;

$sockudo = new Sockudo([
    'app_id' => 'app-id',
    'key' => 'app-key',
    'secret' => 'app-secret',
    'host' => '127.0.0.1',
    'port' => 6001,
    'useTLS' => false,
]);

$sockudo->trigger('orders', 'order.updated', ['id' => 42]);

// With idempotency key
$sockudo->trigger('orders', 'order.updated', ['id' => 42], [], null, [
    'idempotency_key' => 'unique-request-id-123'
]);
```

```go [Go]

import sockudo "github.com/sockudo/sockudo-http-go/v5"

        AppID:   "app-id",
        Key:     "app-key",
        Secret:  "app-secret",
        Host:    "127.0.0.1",
        Port:    "6001",
        Secure:  false,
    }
    s, _ := sockudo.New(client)

    s.Trigger("orders", "order.updated", map[string]interface{}{
        "id": 42,
    })
}
```

## Auth Endpoint Helpers

All server SDKs provide auth helpers for private/presence channels:

::code-group

```js [Node.js]
app.post("/sockudo/auth", (req, res) => {
  const { socket_id, channel_name } = req.body;

  if (channel_name.startsWith("presence-")) {
    const presenceData = {
      user_id: "user-123",
      user_info: { name: "Alice" },
    };
    return res.send(sockudo.authorizeChannel(socket_id, channel_name, presenceData));
  }

  return res.send(sockudo.authorizeChannel(socket_id, channel_name));
});
```

```php [PHP]
// Laravel / Symfony example
Route::post('/sockudo/auth', function (Request $request) use ($sockudo) {
    $socketId    = $request->input('socket_id');
    $channelName = $request->input('channel_name');

    if (str_starts_with($channelName, 'presence-')) {
        $presenceData = ['user_id' => 'user-123', 'user_info' => ['name' => 'Alice']];
        return response()->json($sockudo->authorizeChannel($socketId, $channelName, $presenceData));
    }

    return response()->json($sockudo->authorizeChannel($socketId, $channelName));
});
```

```go [Go]
http.HandleFunc("/sockudo/auth", func(w http.ResponseWriter, r *http.Request) {
    socketID   := r.FormValue("socket_id")
    channelName := r.FormValue("channel_name")

    var response []byte
    if strings.HasPrefix(channelName, "presence-") {
        userData := sockudo.MemberData{
            UserID:   "user-123",
            UserInfo: map[string]string{"name": "Alice"},
        }
        response, _ = client.AuthorizeChannel(socketID, channelName, userData)
    } else {
        response, _ = client.AuthorizeChannel(socketID, channelName, nil)
    }

    w.Header().Set("Content-Type", "application/json")
    w.Write(response)
})
```

```ruby [Ruby]
post '/sockudo/auth' do
  socket_id    = params['socket_id']
  channel_name = params['channel_name']

  if channel_name.start_with?('presence-')
    user_data = { user_id: 'user-123', user_info: { name: 'Alice' } }
    sockudo.authenticate(socket_id, channel_name, user_data).to_json
  else
    sockudo.authenticate(socket_id, channel_name).to_json
  end
end
```

```python [Python]
from flask import Flask, request, jsonify

app = Flask(__name__)

@app.route("/sockudo/auth", methods=["POST"])
def auth():
    socket_id    = request.form.get("socket_id")
    channel_name = request.form.get("channel_name")

    if channel_name.startswith("presence-"):
        user_data = {"user_id": "user-123", "user_info": {"name": "Alice"}}
        return jsonify(client.authenticate(socket_id, channel_name, user_data))

    return jsonify(client.authenticate(socket_id, channel_name))
```

::

The auth signature format is identical to Pusher's HMAC-SHA256 scheme. Auth works the same regardless of protocol version (V1 or V2).

## Webhooks

Verify webhook signatures with the `X-Sockudo-Signature` header against the raw body and app secret:

::code-group

```js [Node.js]
app.post("/webhooks/sockudo", (req, res) => {
  const isValid = sockudo.webhook(req.headers, req.rawBody);
  if (!isValid) return res.sendStatus(401);
  // process webhook payload
  res.sendStatus(200);
});
```

```php [PHP]
Route::post('/webhooks/sockudo', function (Request $request) use ($sockudo) {
    $isValid = $sockudo->verifyWebhookSignature(
        $request->header('X-Sockudo-Signature'),
        $request->getContent()
    );
    if (!$isValid) return response('Unauthorized', 401);
    // process webhook payload
    return response('OK', 200);
});
```

```go [Go]
http.HandleFunc("/webhooks/sockudo", func(w http.ResponseWriter, r *http.Request) {
    body, _ := io.ReadAll(r.Body)
    signature := r.Header.Get("X-Sockudo-Signature")

    if !client.VerifyWebhookSignature(signature, body) {
        http.Error(w, "Unauthorized", http.StatusUnauthorized)
        return
    }
    // process webhook payload
    w.WriteHeader(http.StatusOK)
})
```

```ruby [Ruby]
post '/webhooks/sockudo' do
  signature = request.env['HTTP_X_SOCKUDO_SIGNATURE']
  body      = request.body.read

  unless sockudo.verify_webhook_signature(signature, body)
    halt 401, 'Unauthorized'
  end
  # process webhook payload
  status 200
end
```

```python [Python]
@app.route("/webhooks/sockudo", methods=["POST"])
def webhook():
    signature = request.headers.get("X-Sockudo-Signature")
    body      = request.get_data()

    if not client.verify_webhook_signature(signature, body):
        return "Unauthorized", 401
    # process webhook payload
    return "OK", 200
```

## Idempotency Key

All Sockudo server SDKs support `idempotency_key` for safe publish retries. When provided, the server ensures the event is broadcast only once even if the HTTP request is retried:

- Pass `idempotency_key: "unique-id"` as a string for an explicit key
- Pass `idempotency_key: true` to auto-generate a UUIDv4

The server caches keys and returns the same response for duplicates.

## Pusher Server SDK Compatibility

If you already use Pusher server SDKs, they work with Sockudo's HTTP API by changing the host/port:

```js
// Using the official Pusher Node.js SDK
const Pusher = require("pusher");

const pusher = new Pusher({
  appId: "app-id",
  key: "app-key",
  secret: "app-secret",
  host: "127.0.0.1",
  port: "6001",
  useTLS: false,
  cluster: "mt1",
});

await pusher.trigger("orders", "order.updated", { id: 42 });
```

Pusher server SDKs do not support `idempotency_key`. For that feature, use the Sockudo server SDKs.

For migration planning, treat the compatibility levels as:

- **Protocol-compatible**: Sockudo accepts the same signed HTTP API/auth/webhook model.
- **Source-compatible**: existing application code can often be reused with only configuration changes.
- **Package-compatible**: not guaranteed across every language, because Sockudo SDKs use their own package/module identities.

## SDK-Agnostic Fallback

Call the Sockudo HTTP API directly with signed query params:

- `POST /apps/{appId}/events`
- `POST /apps/{appId}/batch_events`

See [HTTP API](/server/http-api) for details.
