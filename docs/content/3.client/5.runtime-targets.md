## Browser

```ts
import { SockudoClient } from "@sockudo/client";
```

Default export resolves to web build in browser environments.

## Node.js

```ts
import { SockudoClient } from "@sockudo/client";
```

Node target uses native modern APIs (`WebSocket`, `fetch`-first auth/timeline flow).

## Web Worker

```ts
import SockudoWorker from "@sockudo/client/worker";
```

Use worker entrypoint to avoid incorrect runtime shims.

## React

```ts
import { SockudoProvider, useChannel } from "@sockudo/client/react";
```

The React entrypoint is a thin adapter over the same protocol V2 client. It does not change transport or protocol behavior.

## Vue

```ts
import { createSockudoPlugin, useChannel } from "@sockudo/client/vue";
```

The Vue entrypoint exposes composables and DI helpers over the same `@sockudo/client` runtime.

## React Native

```ts
import SockudoReactNative from "@sockudo/client/react-native";
```

Optional peer dependency:

```bash
npm install @react-native-community/netinfo
```

## Encryption Builds

- `@sockudo/client/with-encryption`
- `@sockudo/client/worker/with-encryption`

Use these for `private-encrypted-*` channel workflows.

## Swift

- Package: `sockudo-swift`
- Platforms: iOS, macOS, tvOS, watchOS
- Distribution: Swift Package Manager

```swift
import SockudoSwift
```

## Kotlin

- Package: `sockudo-kotlin`
- Platforms: Android and JVM
- Distribution: Maven Central style Gradle dependency

```kotlin
import io.sockudo.client.SockudoClient
```

## Flutter / Dart

- Package: `sockudo_flutter`
- Platforms: Flutter mobile/desktop and Dart
- Distribution: `pub.dev`

```dart
import 'package:sockudo_flutter/sockudo_flutter.dart';
```
