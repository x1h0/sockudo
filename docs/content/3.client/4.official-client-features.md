## V2 Features in Official Sockudo Clients

Official Sockudo client SDKs default to protocol V2 (`?protocol=2`), which provides:

- `sockudo:` event prefix (Sockudo-native protocol)
- Serial number tracking on every message
- `message_id` for deduplication
- Connection recovery via `sockudo:resume`
- Filter-aware subscription API
- Automatic delta decoding
- Per-subscription delta negotiation
- Delta stats hooks and error hooks
- Encrypted channel shared-secret auth and payload decryption
- Wire-format negotiation for `json`, `messagepack`, and `protobuf`
- Wildcard channel subscriptions and `[meta]channel` metachannels

Set `protocolVersion: 1` to use Pusher-compatible mode with `pusher:` prefixes.

## Tag Filtering Subscribe API

::code-group

```js [JavaScript]
import { SockudoClient } from "@sockudo/client";
import { Filter } from "@sockudo/client/filter";

const client = new SockudoClient("app-key", {
  wsHost: "127.0.0.1",
  wsPort: 6001,
  forceTLS: false,
});

const channel = client.subscribe("market-data", {
  filter: Filter.and(
    Filter.eq("event_type", "trade"),
    Filter.gte("price", "100")
  ),
});
```

```swift [Swift]
let channel = client.subscribe(
    "market-data",
    options: .init(
        filter: .and([
            .eq("event_type", "trade"),
            .gte("price", "100")
        ])
    )
)
```

```kotlin [Kotlin]
val channel = client.subscribe(
    "market-data",
    SubscriptionOptions(
        filter = Filter.and(
            Filter.eq("event_type", "trade"),
            Filter.gte("price", "100"),
        ),
    ),
)
```

```dart [Flutter]
final channel = client.subscribe(
  'market-data',
  options: SubscriptionOptions(
    filter: Filter.and(<FilterNode>[
      Filter.eq('event_type', 'trade'),
      Filter.gte('price', '100'),
    ]),
  ),
);
```

## Delta Compression (Global Enable)

::code-group

```js [JavaScript]
const client = new SockudoClient("app-key", {
  wsHost: "127.0.0.1",
  wsPort: 6001,
  forceTLS: false,
  deltaCompression: {
    enabled: true,
    algorithms: ["fossil", "xdelta3"],
    debug: false,
    onStats: (stats) => console.log(stats.bandwidthSavedPercent),
  },
});
```

```swift [Swift]
let client = try SockudoClient(
    "app-key",
    options: .init(
        cluster: "local",
        forceTLS: false,
        wsHost: "127.0.0.1",
        wsPort: 6001,
        deltaCompression: .init(
            enabled: true,
            algorithms: [.fossil, .xdelta3],
            debug: false
        )
    )
)
```

```kotlin [Kotlin]
val client = SockudoClient(
    "app-key",
    SockudoOptions(
        cluster = "local",
        forceTls = false,
        wsHost = "127.0.0.1",
        wsPort = 6001,
        deltaCompression = DeltaOptions(
            enabled = true,
            algorithms = listOf(DeltaAlgorithm.fossil, DeltaAlgorithm.xdelta3),
            debug = false,
        ),
    ),
)
```

```dart [Flutter]
final client = SockudoClient(
  'app-key',
  const SockudoOptions(
    cluster: 'local',
    forceTls: false,
    wsHost: '127.0.0.1',
    wsPort: 6001,
    deltaCompression: DeltaOptions(
      enabled: true,
      algorithms: <DeltaAlgorithm>[
        DeltaAlgorithm.fossil,
        DeltaAlgorithm.xdelta3,
      ],
      debug: false,
    ),
  ),
);
```

## Per-Subscription Delta Options

::code-group

```js [JavaScript]
// Delta with algorithm
client.subscribe("ticker:BTC", {
  delta: { enabled: true, algorithm: "fossil" },
});

// Filter + delta together
client.subscribe("events", {
  filter: Filter.eq("priority", "high"),
  delta: { algorithm: "xdelta3" },
});
```

```swift [Swift]
let channel = client.subscribe(
    "events",
    options: .init(
        filter: .eq("priority", "high"),
        delta: .init(algorithm: .xdelta3)
    )
)
```

```kotlin [Kotlin]
val channel = client.subscribe(
    "events",
    SubscriptionOptions(
        filter = Filter.eq("priority", "high"),
        delta = ChannelDeltaSettings(algorithm = DeltaAlgorithm.xdelta3),
    ),
)
```

```dart [Flutter]
final channel = client.subscribe(
  'events',
  options: const SubscriptionOptions(
    filter: FilterNode(key: 'priority', cmp: 'eq', val: 'high'),
    delta: ChannelDeltaSettings(algorithm: DeltaAlgorithm.xdelta3),
  ),
);
```

## Conflation Key Support

All official clients automatically handle conflation keys when the server enables them. The client:

- Receives delta cache sync messages with entity-grouped states
- Maintains separate delta histories per conflation key (e.g., per stock symbol, per device ID)
- Reconstructs delta messages using the correct entity's base message
- Tracks cache stats per conflation key when `debug: true`

No additional client configuration needed — conflation is configured server-side and the client adapts automatically.

Example server config:

```toml
[app_manager.array.apps.policy.channels.channel_delta_compression."market:*"]
enabled = true
conflation_key = "symbol"
max_messages_per_key = 100
max_conflation_keys = 1000
```

See [Delta Compression](/server/delta-compression#conflation-keys) for server configuration details.

## Pusher Clients (V1 only)

Core channels/events work on plain `pusher-js` and Laravel Echo using protocol V1.
They do not support V2 features (delta, tag filtering, serial, message_id, connection recovery).
