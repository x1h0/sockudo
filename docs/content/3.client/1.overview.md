Sockudo ships four official client SDKs:

| Client | Package | Runtimes |
|---|---|---|
| JavaScript / TypeScript | `@sockudo/client` | Browser, Node.js, Workers, React hooks, Vue composables, React Native |
| Swift | `SockudoSwift` (SPM) | iOS, macOS, tvOS, watchOS |
| Kotlin | `io.sockudo:sockudo-kotlin` | Android, JVM |
| Flutter / Dart | `sockudo_flutter` | Flutter mobile, desktop, Dart |

## Protocol Version

All Sockudo client SDKs default to **protocol V2** (`?protocol=2`), which provides:

- `sockudo:` event prefix (Sockudo-native)
- Serial number tracking on every message
- `message_id` for deduplication
- Connection recovery via `sockudo:resume`
- Delta compression and tag filtering

Set `protocolVersion: 1` for Pusher-compatible mode (`pusher:` prefix, no V2 features). For V1 you can also use official Pusher SDKs (`pusher-js`, etc.) directly.

## Core Features (all protocols)

- `subscribe`, `unsubscribe`, `bind`, `unbind`
- Public, private, presence, and encrypted channels
- Private/presence auth endpoint flow
- Client events (`client-*`)
- User sign-in and watchlist events

## V2 Features

- Filter-aware subscribe options (server-side tag filtering)
- Delta compression negotiation and reconstruction (Fossil, Xdelta3/VCDIFF)
- Per-channel delta settings
- Serial tracking and connection recovery
- `message_id` deduplication
- Wire-format negotiation: `json`, `messagepack`, and `protobuf`
- Wildcard channel subscriptions and metachannels
- Framework adapters in-package:
  - `@sockudo/client/react`
  - `@sockudo/client/vue`

## Pusher Compatibility

For users who only need core Pusher behavior, `pusher-js` and Laravel Echo work with Sockudo using protocol V1. They do not support V2 features (delta, tag filtering, recovery). See [pusher-js integration](/integrations/pusher-js).
