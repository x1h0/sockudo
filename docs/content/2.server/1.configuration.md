## Config Sources and Priority

Sockudo applies configuration in this order:

1. Internal defaults.
2. `config/config.toml` (if present).
3. `config/config.json` (if present, fallback).
4. CLI `--config` file.
5. Environment variables (highest priority).

## Core Config Shape

Main config object groups:

| Section | Purpose |
|---|---|
| `adapter` | Pub/sub transport (`local`, `redis`, `redis-cluster`, `nats`, `rabbitmq`, `google-pubsub`, `kafka`) |
| `app_manager` | App credential source (`memory`, `mysql`, `postgres`, `dynamodb`, `surrealdb`, `scylladb`) |
| `cache` | Cache backend (`memory`, `redis`, `redis-cluster`, `none`) |
| `queue` | Queue backend (`memory`, `redis`, `redis-cluster`, `sqs`, `none`) |
| `rate_limiter` | Rate limiting backend and rules |
| `database` | MySQL, PostgreSQL, Redis, DynamoDB, SurrealDB, ScyllaDB connection settings |
| `database_pooling` | Global connection pool settings |
| `metrics` | Prometheus metrics endpoint |
| `cors` | Cross-origin resource sharing |
| `ssl` | TLS termination |
| `unix_socket` | Unix domain socket listener |
| `webhooks` | Webhook delivery and batching |
| `cleanup` | Async disconnect processing |
| `cluster_health` | Multi-node health monitoring |
| `delta_compression` | Delta compression for repeated messages |
| `tag_filtering` | Tag-based message filtering |
| `websocket` | WebSocket buffer and protocol settings |
| `channel_limits` | Channel name and cache limits |
| `event_limits` | Event size and batch limits |
| `presence` | Presence channel limits |
| `http_api` | HTTP API request limits |
| `logging` | Log output formatting |

Minimal starter:

```toml [config/config.toml]
host = "0.0.0.0"
port = 6001

[adapter]
driver = "local"

[app_manager]
driver = "memory"

[app_manager.array]
[[app_manager.array.apps]]
id = "app-id"
key = "app-key"
secret = "app-secret"
enabled = true
[app_manager.array.apps.policy.limits]
max_connections = 100000
max_client_events_per_second = 1000

[app_manager.array.apps.policy.features]
enable_client_messages = true

[app_manager.array.apps.policy.channels]
allowed_origins = ["*"]

[[app_manager.array.apps.policy.channels.channel_namespaces]]
name = "ticker"
channel_name_pattern = "^ticker:[A-Za-z0-9._-]+$"
max_channel_name_length = 200

[[app_manager.array.apps.policy.channels.channel_namespaces]]
name = "chat"
channel_name_pattern = "^chat:[A-Za-z0-9._-]+(?:#[A-Za-z0-9._-]+)?$"
allow_user_limited_channels = true
```

## App-Level Limits

Per-app settings live under `app.policy.*` and control behavior and protection boundaries:

| Setting | Default | Description |
|---|---|---|
| `policy.limits.max_connections` | `100` | Max simultaneous connections |
| `policy.limits.max_client_events_per_second` | `100` | Client event rate |
| `policy.limits.max_read_requests_per_second` | `100` | Read request rate |
| `policy.limits.max_backend_events_per_second` | `100` | Backend event rate |
| `policy.limits.max_presence_members_per_channel` | `100` | Max presence members |
| `policy.limits.max_presence_member_size_in_kb` | `100` | Max presence member data |
| `policy.limits.max_channel_name_length` | `100` | Max channel name length |
| `policy.limits.max_event_channels_at_once` | `100` | Max channels per broadcast |
| `policy.limits.max_event_name_length` | `100` | Max event name length |
| `policy.limits.max_event_payload_in_kb` | `100` | Max event payload |
| `policy.limits.max_event_batch_size` | `100` | Max events per batch |
| `policy.features.enable_client_messages` | `false` | Allow client-to-client events |
| `policy.features.enable_user_authentication` | `false` | Require user auth |
| `policy.features.enable_watchlist_events` | `false` | Enable watchlist events |
| `policy.channels.allowed_origins` | `null` | Restrict WebSocket origins |
| `policy.channels.channel_namespaces` | `null` | Namespace-scoped channel rules for V2 |
| `policy.channels.channel_delta_compression` | `null` | Per-channel delta config |
| `policy.idempotency` | `null` | Per-app override for HTTP/message dedup |
| `policy.connection_recovery` | `null` | Per-app override for V2 recovery |

User-limited channels use the `namespace:channel#user_id` form and must belong to a namespace with `allow_user_limited_channels = true`.

## CLI Configuration File Override

```bash [Terminal]
./sockudo --config ./config/production.toml
```

or:

```bash [Terminal]
cargo run --release -- --config ./config/production.toml
```

## Defaults Worth Knowing

| Setting | Default |
|---|---|
| HTTP/WebSocket port | `6001` |
| Metrics port | `9601` |
| Activity timeout | `120s` |
| Shutdown grace period | `10s` |
| User auth timeout | `3600s` |
| WebSocket max payload | `64 KB` |
| Delta compression | enabled (`fossil`) |
| Tag filtering | disabled |
| WebSocket buffer | `max_messages=1000` |
| Rate limiter | enabled (memory backend) |
| Database pooling | enabled (min=2, max=10) |
| Webhook batching | enabled (50ms window) |
| Async cleanup | enabled |
| Cluster health | enabled |

See [Config Reference](/reference/config-reference) for the complete JSON structure and [Environment Variables](/server/environment-variables) for all env var overrides.
