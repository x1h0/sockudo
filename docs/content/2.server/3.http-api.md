## Endpoints

### Realtime Publish

- `POST /apps/{appId}/events` — publish a single event to one or more channels
- `POST /apps/{appId}/batch_events` — publish multiple events in one request

### Channel Queries

- `GET /apps/{appId}/channels` — list occupied channels, with optional prefix filter
- `GET /apps/{appId}/channels/{channelName}` — get info for a specific channel
- `GET /apps/{appId}/channels/{channelName}/users` — list members of a presence channel

### User Management

- `POST /apps/{appId}/users/{userId}/terminate_connections` — disconnect all sockets for a user

### Health and Ops

- `GET /up` — server health check
- `GET /up/{appId}` — per-app health check
- `GET /usage` — memory usage (can be disabled via `http_api.usage_enabled: false`)
- `GET /stats` — server and per-app connection statistics (same gate as `/usage`)
- `GET /metrics` — Prometheus metrics (served on the metrics port, default `9601`)

## Publish Single Event

`POST /apps/{appId}/events`

Broadcasts an event to one or more channels.

**Request body**

| Field | Type | Required | Description |
|---|---|---|---|
| `name` | string | Yes | Event name (e.g. `"my-event"`) |
| `data` | string | Yes | Event payload. Must be a JSON-encoded string; max size enforced by `max_event_payload_in_kb` |
| `channel` | string | One of | Target channel name. Use either `channel` or `channels`, not both |
| `channels` | string[] | One of | List of target channels (max 100 by default) |
| `socket_id` | string | No | Socket to exclude from delivery |
| `info` | string | No | Comma-separated list of info fields to return: `subscription_count`, `user_count`, `cache` |
| `tags` | object | No | Key-value tags for server-side tag filtering (V2 connections only) |
| `idempotency_key` | string | No | Deduplication key; duplicate publishes with the same key within the TTL window are ignored |

**Response** `200 OK`

If `info` was not requested:

```json
{}
```

If `info` was requested, the response contains per-channel stats:

```json
{
  "channels": {
    "my-channel": {
      "subscription_count": 10,
      "user_count": 5
    }
  }
}
```

`user_count` is only populated for presence channels. `cache` returns the last cached payload for cache channels.

**Error responses**

| Status | Reason |
|---|---|
| `400` | Missing required field, invalid JSON, `channel` and `channels` both set, too many channels, or `user_count` requested on a non-presence channel |
| `401` | Invalid or missing auth signature |
| `413` | Event payload exceeds the configured size limit |

## Publish Batch Events

`POST /apps/{appId}/batch_events`

Publishes multiple events in a single request. Each item in the `batch` array follows the same shape as a single event publish. The max batch size is configurable (`max_batch_messages`, default 10).

**Request body**

```json
{
  "batch": [
    {
      "name": "event-one",
      "channel": "ch-a",
      "data": "{\"hello\":\"world\"}",
      "info": "subscription_count",
      "idempotency_key": "key-1"
    },
    {
      "name": "event-two",
      "channels": ["ch-b", "ch-c"],
      "data": "{\"foo\":\"bar\"}",
      "idempotency_key": "key-2"
    }
  ]
}
```

**Response** `200 OK`

Returns a `batch` array in the same order. Each entry is an object with the requested `info` fields, or an empty object if no info was requested.

```json
{
  "batch": [
    { "subscription_count": 5 },
    {}
  ]
}
```

**Error responses** — same codes as the single-event endpoint; a single invalid item rejects the entire request.

## List Channels

`GET /apps/{appId}/channels`

Returns all occupied channels for the app.

**Query parameters**

| Parameter | Description |
|---|---|
| `filter_by_prefix` | Only return channels whose name starts with this value (e.g. `presence-`) |
| `info` | Comma-separated info fields: `subscription_count`, `user_count` |

`user_count` is only available when `filter_by_prefix=presence-`.

**Response** `200 OK`

```json
{
  "channels": {
    "my-channel": {
      "subscription_count": 4
    },
    "presence-room": {
      "subscription_count": 3,
      "user_count": 3
    }
  }
}
```

Channels with no subscribers are not included.

## Get Channel Info

`GET /apps/{appId}/channels/{channelName}`

**Query parameters**

| Parameter | Description |
|---|---|
| `info` | Comma-separated: `subscription_count`, `user_count`, `cache` |

`user_count` is only valid for presence channels (name must start with `presence-`). Requesting it on any other channel type returns `400`.

**Response** `200 OK`

```json
{
  "occupied": true,
  "subscription_count": 10,
  "user_count": 5,
  "cache": "{\"last\":\"payload\"}"
}
```

`cache` is only populated for cache channels and only when explicitly requested via `info=cache`.

## Get Presence Channel Users

`GET /apps/{appId}/channels/{channelName}/users`

Only works on presence channels (name must start with `presence-`).

**Response** `200 OK`

```json
{
  "users": [
    { "id": "user-1" },
    { "id": "user-2" }
  ]
}
```

Returns `400` if the channel is not a presence channel.

## Terminate User Connections

`POST /apps/{appId}/users/{userId}/terminate_connections`

Disconnects all WebSocket connections belonging to the given user ID. The user ID is the one supplied in the presence channel auth payload.

**Request body** — empty (no body required)

**Response** `200 OK`

```json
{}
```

## Health Checks

### `GET /up`

General server health. No authentication required.

Checks the adapter, cache manager, and queue/webhook system. Non-critical subsystems (queue/webhooks) cause a `degraded` state rather than an error.

| Status | HTTP code | `X-Health-Check` header |
|---|---|---|
| All systems healthy | `200` | `ok` |
| Non-critical system degraded | `304` | `degraded` |
| Critical system unhealthy | `503` | `error` |

### `GET /up/{appId}`

Same checks as `/up`, plus validates that the app exists and is enabled. Returns `404` if the app is not found or is disabled.

## Signed Query Parameters

All routes under `/apps/*` require Pusher-style query auth:

- `auth_key` — app key
- `auth_timestamp` — Unix timestamp (seconds)
- `auth_version` — always `"1.0"`
- `auth_signature` — HMAC-SHA256 hex digest
- `body_md5` — MD5 hex digest of the request body (required for non-empty `POST` bodies)

The timestamp must be within ±10 minutes of the server clock.

## Signature Format

String to sign:

```text
{METHOD}\n{REQUEST_PATH}\n{SORTED_QUERY_EXCLUDING_AUTH_SIGNATURE}
```

Query params are sorted alphabetically by key and joined with `&`. `auth_signature` itself is excluded from the string before signing.

## Signing Example

::code-group

```ts [Node.js]
import crypto from "node:crypto";

export function signPusherRequest({
  method,
  path,
  appKey,
  appSecret,
  body,
}: {
  method: "GET" | "POST";
  path: string;
  appKey: string;
  appSecret: string;
  body?: string;
}) {
  const auth_timestamp = Math.floor(Date.now() / 1000).toString();
  const params: Record<string, string> = {
    auth_key: appKey,
    auth_timestamp,
    auth_version: "1.0",
  };

  if (method === "POST" && body && body.length > 0) {
    params.body_md5 = crypto.createHash("md5").update(body).digest("hex");
  }

  const queryForSig = Object.keys(params)
    .sort()
    .map((k) => `${k}=${params[k]}`)
    .join("&");

  const stringToSign = `${method}\n${path}\n${queryForSig}`;
  const auth_signature = crypto
    .createHmac("sha256", appSecret)
    .update(stringToSign)
    .digest("hex");

  return { ...params, auth_signature };
}
```

```go [Go]
package main

import (
    "crypto/hmac"
    "crypto/md5"
    "crypto/sha256"
    "encoding/hex"
    "fmt"
    "net/url"
    "sort"
    "strconv"
    "strings"
    "time"
)

func signRequest(method, path, appKey, appSecret, body string) url.Values {
    params := url.Values{}
    params.Set("auth_key", appKey)
    params.Set("auth_timestamp", strconv.FormatInt(time.Now().Unix(), 10))
    params.Set("auth_version", "1.0")

    if method == "POST" && body != "" {
        h := md5.Sum([]byte(body))
        params.Set("body_md5", hex.EncodeToString(h[:]))
    }

    keys := make([]string, 0, len(params))
    for k := range params {
        keys = append(keys, k)
    }
    sort.Strings(keys)
    parts := make([]string, 0, len(keys))
    for _, k := range keys {
        parts = append(parts, k+"="+params.Get(k))
    }
    queryForSig := strings.Join(parts, "&")

    stringToSign := fmt.Sprintf("%s\n%s\n%s", method, path, queryForSig)
    mac := hmac.New(sha256.New, []byte(appSecret))
    mac.Write([]byte(stringToSign))
    params.Set("auth_signature", hex.EncodeToString(mac.Sum(nil)))
    return params
}
```

```php [PHP]
function signRequest(string $method, string $path, string $appKey, string $appSecret, string $body = ''): array {
    $params = [
        'auth_key'       => $appKey,
        'auth_timestamp' => time(),
        'auth_version'   => '1.0',
    ];

    if ($method === 'POST' && $body !== '') {
        $params['body_md5'] = md5($body);
    }

    ksort($params);
    $queryForSig = http_build_query($params);
    $stringToSign = implode("\n", [$method, $path, $queryForSig]);
    $params['auth_signature'] = hash_hmac('sha256', $stringToSign, $appSecret);

    return $params;
}
```

```python [Python]
import hashlib
import hmac
import time
from urllib.parse import urlencode

def sign_request(method: str, path: str, app_key: str, app_secret: str, body: str = "") -> dict:
    params = {
        "auth_key": app_key,
        "auth_timestamp": str(int(time.time())),
        "auth_version": "1.0",
    }

    if method == "POST" and body:
        params["body_md5"] = hashlib.md5(body.encode()).hexdigest()

    query_for_sig = urlencode(sorted(params.items()))
    string_to_sign = f"{method}\n{path}\n{query_for_sig}"
    signature = hmac.new(app_secret.encode(), string_to_sign.encode(), hashlib.sha256).hexdigest()
    params["auth_signature"] = signature
    return params
```

```ruby [Ruby]
require 'openssl'
require 'digest'

def sign_request(method, path, app_key, app_secret, body = '')
  params = {
    'auth_key'       => app_key,
    'auth_timestamp' => Time.now.to_i.to_s,
    'auth_version'   => '1.0',
  }

  params['body_md5'] = Digest::MD5.hexdigest(body) if method == 'POST' && !body.empty?

  query_for_sig = params.sort.map { |k, v| "#{k}=#{v}" }.join('&')
  string_to_sign = "#{method}\n#{path}\n#{query_for_sig}"
  auth_signature = OpenSSL::HMAC.hexdigest('sha256', app_secret, string_to_sign)

  params.merge('auth_signature' => auth_signature)
end
```

## Common Query Patterns

```text
# List only presence channels with member counts
GET /apps/{appId}/channels?filter_by_prefix=presence-&info=user_count

# Get subscription and member count for one channel
GET /apps/{appId}/channels/{name}?info=subscription_count,user_count

# Get cached payload for a cache channel
GET /apps/{appId}/channels/{name}?info=cache

# Publish and get back subscriber count
POST /apps/{appId}/events?...&info=subscription_count
```
