## Overview

The **extras envelope** is a structured metadata object that can accompany any V2 message payload. It provides a uniform way to attach out-of-band information to events without modifying the event data itself.

::callout{type="info"}
Extras is a **protocol V2 only** feature. V1 (Pusher-compatible) connections receive messages with the extras field stripped entirely, ensuring backward compatibility with official Pusher SDKs.
::

## Extras Fields

The extras object supports four fields:

| Field | Type | Purpose |
|---|---|---|
| `headers` | `object` | Arbitrary key-value metadata forwarded to subscribers |
| `ephemeral` | `bool` | Mark message as fire-and-forget (skip recovery buffer and webhooks) |
| `idempotency_key` | `string` | Server-side deduplication key for publish retries |
| `echo` | `bool` | Per-message override for echo control (whether sender receives own message) |

All fields are optional. An empty extras object is equivalent to omitting it.

## V1 vs V2 Behavior

| | V1 Connection | V2 Connection |
|---|---|---|
| **Extras received** | Never (stripped before delivery) | Always (when present) |
| **Headers forwarded** | No | Yes |
| **Ephemeral respected** | Message still delivered, but extras stripped | Fully supported |
| **Echo control** | Always receives own messages | Respects `extras.echo` |

V1 connections are never affected by extras. The server strips the extras field before delivering to V1 subscribers, so existing Pusher SDK integrations continue to work without changes.

## HTTP API Usage

Include an `extras` object in the publish request body:

```json
POST /apps/{app_id}/events
{
  "name": "price-update",
  "channel": "market-data",
  "data": "{\"symbol\":\"AAPL\",\"price\":182.50}",
  "extras": {
    "headers": {
      "region": "us-east",
      "priority": "high"
    },
    "ephemeral": false,
    "idempotency_key": "price-AAPL-1706123456",
    "echo": true
  }
}
```

### Batch Publishing

Extras can be set per-event in batch publishes:

```json
POST /apps/{app_id}/batch_events
{
  "batch": [
    {
      "name": "cursor-move",
      "channel": "collaboration-room",
      "data": "{\"x\":120,\"y\":340}",
      "extras": {
        "ephemeral": true
      }
    },
    {
      "name": "document-save",
      "channel": "collaboration-room",
      "data": "{\"version\":42}",
      "extras": {
        "idempotency_key": "save-v42"
      }
    }
  ]
}
```

## Client SDK Usage

V2 client SDKs can pass extras when triggering client events:

```javascript
channel.trigger('client-typing', { user: 'alice' }, {
  extras: {
    ephemeral: true,
    echo: false
  }
});
```

Received events on V2 connections include extras when present:

```javascript
channel.bind('price-update', (data, metadata) => {
  console.log(data);              // { symbol: "AAPL", price: 182.50 }
  console.log(metadata.extras);   // { headers: { region: "us-east" } }
});
```

## Message Format

A V2 message with extras looks like this on the wire:

```json
{
  "event": "sockudo:price-update",
  "channel": "market-data",
  "data": "{\"symbol\":\"AAPL\",\"price\":182.50}",
  "serial": 47,
  "message_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "extras": {
    "headers": {
      "region": "us-east",
      "priority": "high"
    }
  }
}
```

The same message delivered to a V1 connection:

```json
{
  "event": "price-update",
  "channel": "market-data",
  "data": "{\"symbol\":\"AAPL\",\"price\":182.50}"
}
```

No `serial`, `message_id`, or `extras` fields are present on V1.
