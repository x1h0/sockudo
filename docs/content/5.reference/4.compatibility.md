## Protocol Version Support

| Client | Protocol V1 | Protocol V2 | Default |
| --- | --- | --- | --- |
| `pusher-js` | Yes | No | V1 |
| Laravel Echo | Yes | No | V1 |
| `@sockudo/client` (JS) | Yes | Yes | V2 |
| `sockudo-swift` | Yes | Yes | V2 |
| `sockudo-kotlin` | Yes | Yes | V2 |
| `sockudo-flutter` | Yes | Yes | V2 |

## Client Feature Compatibility

| Feature | pusher-js | Laravel Echo | `@sockudo/client` |
| --- | --- | --- | --- |
| Public/private/presence channels | Yes | Yes | Yes |
| Channel auth endpoint flow | Yes | Yes | Yes |
| User signin flow | Yes (manual wiring) | Partial | Yes |
| Client events (`client-*`) | Yes | Yes | Yes |
| Wire formats (`json`) | Yes | Yes | Yes |
| Wire formats (`messagepack`, `protobuf`) | No | No | Yes (V2) |
| Tag filter subscribe API | No | No | Yes (V2) |
| Automatic delta decode | No | No | Yes (V2) |
| Per-subscription delta options | No | No | Yes (V2) |
| Serial number tracking | No | No | Yes (V2) |
| `message_id` deduplication | No | No | Yes (V2) |
| Connection recovery (`sockudo:resume`) | No | No | Yes (V2) |
| Wildcard channel subscriptions | No | No | Yes (V2) |
| Metachannels (`[meta]channel`) | No | No | Yes (V2) |

## Server SDK Compatibility

| Capability | Sockudo server SDKs | Pusher server SDKs |
| --- | --- | --- |
| Event trigger | Native | Works (set Sockudo `host`/`port`) |
| Channel auth generation | Native | Works (`authorizeChannel`) |
| Webhook signature model | Same HMAC model | Same HMAC model |
| `idempotency_key` support | Yes | No |

## Migration Guidance

- **Stay on pusher-js / Echo** if you only need core Pusher behavior (V1 protocol).
- **Use `@sockudo/client`** to adopt protocol V2 features: connection recovery, serial numbers, message IDs, tag filtering, and delta compression.
- **Gradual migration**: the same Sockudo server handles both V1 and V2 connections simultaneously. Upgrade clients one at a time.
- **Server SDKs**: Sockudo server SDKs (`sockudo`, `sockudo-http-go`, etc.) are recommended for `idempotency_key` support. Pusher server SDKs still work for basic event triggering and channel auth.
